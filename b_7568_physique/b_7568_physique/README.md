# BOJ 7568 - 덩치

## Idea
- 각 사람 `i`의 등수(덩치 순위)는 **(몸무게, 키)**가 모두 더 큰 사람의 수에 의해 결정된다.
- 사람 `i`보다 덩치가 큰 사람 `j`의 조건은 다음 한 줄:
  - `weight[i] < weight[j] && height[i] < height[j]`
- `i`보다 덩치 큰 사람이 `k`명이면, `i`의 등수는 **`k + 1`**.
  - 초기값을 `1`로 두고, 조건을 만족하는 `j`를 만날 때마다 `count[i]++` 하면 된다.
- N(≤50)이 작아서 모든 쌍을 비교하는 **O(N^2)** 브루트포스가 정답.

## Complexity
- Time: **O(N^2)**
  - N: 사람 수(최대 50)
  - 모든 (i, j) 쌍 비교
- Space: **O(N)**
  - `weight[]`, `height[]`, `count[]`

## How to Test
Example 1:
Input:
5
55 185
58 183
88 186
60 175
46 155

Output:
2 2 1 2 5

## Pitfalls
- 비교 조건을 `<=`로 바꾸면 오답:
  - **둘 다 커야** 덩치가 큰 것으로 인정한다.
- `if (i == j) continue;`를 넣어도 무방.
- 출력 형식:
  - 마지막에 공백이 하나 붙어도 BOJ는 보통 허용되지만, 깔끔하게 하려면 마지막 공백 처리를 하거나 `'\n'`을 추가하자.

## Implementation Notes
- `N <= 50`이므로 정렬/좌표압축 같은 복잡한 최적화는 불필요.
- 배열을 정적 크기(50)로 두면 메모리/동작이 예측 가능해서 임베디드 관점에서도 깔끔하다.
- I/O 성능:
  - `ios::sync_with_stdio(false); cin.tie(nullptr);`로 불필요한 동기화/flush를 줄였다.
